문제1. 여러 서버에 Ping 통신 체크 프로그램
다수의 서버에 대해 Ping 테스트를 수행하고, 응답 상태를 기록하는 프로그램을 작성하세요.
프로그램은 서버 목록 파일을 읽고, 서버의 IP 주소와 호스트네임을 확인하여 Ping 통신 상태를 체크합니다.
결과는 파일로 저장하며, 예외 처리와 정규표현식을 활용해 서버 목록의 유효성을 검사합니다.

요구사항
1. 서버 클래스 정의
    • Server 클래스를 작성하고, 다음 속성과 메서드 추가
         • 속성
           - 서버 hostname
           - IP 주소
           - Ping 상태
          • 메서드
                 • 서버의 Ping 테스트를 수행하여 상태를 업데이트.
                  -  성공 시 status를 OK로 설정.
                  -  실패 시 status를 FAIL로 설정.
                  • IP 주소가 유효한 IPv4 형식인지 정규표현식으로 검사.

2. Ping 테스트 관리 클래스 정의
   • PingTester 클래스를 작성하고, 다음 속성과 메서드 추가
          • 속성
           - 서버 목록 파일 경로
           - 서버 리스트
          • 메서드
           - 서버 목록 파일을 읽고, 각 줄에서 서버 이름과 IP 주소를 추출하여 Server 객체로 변환. 정규표현식으로 유효하지 않은 IP 주소나 호스트네임이 있으면 예외를 발생.
           - 서버 리스트에 대해 ping() 메서드를 호출하여 Ping 상태를 업데이트.
           - 모든 서버의 Ping 상태를 요약하여 리포트를 생성하고, 파일로 저장.

3. 리포트 저장
   • 분석 결과를 ping_report.txt 파일에 저장.
   • 리포트는 다음 정보를 포함
      - OK 서버 목록
      - FAIL 서버 목록

4. 예외처리
  • 서버 목록 파일이 존재하지 않거나 잘못된 형식일 경우 예외를 처리하고 오류 메시지 출력.


구현의 핵심 포인트
1. 정규표현식
• IP 주소 유효성 검사.
• 서버 목록 파일의 각 줄에서 서버 이름과 IP 주소 추출.

2. 파일 입출력
• 서버 목록 파일 읽기.
• Ping 테스트 결과를 리포트 파일에 저장.

3. 예외 처리
• 파일 미존재 또는 잘못된 파일 형식 예외 처리.
• 유효하지 않은 IP 주소 예외 처리.

4. 클래스 설계
• Server와 PingTester 클래스의 역할 분리.
• 객체 지향 설계를 활용하여 각 서버에 대한 Ping 테스트를 수행
